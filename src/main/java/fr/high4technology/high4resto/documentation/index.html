Api

L'api est dévellopée avec Spring Reactive, elle se veut non bloquante. La sécurité ce fait avec le protocol JWT HS256, mais dans le cadre du dévellopement Front End nous n'avons pas besoin d'avoir un header spécial

La majorité des éléments peuvent s'obtenir de la même façon avec une simple requete Get, il n'y a que les imformations concernant la manipulation de l'objet Client qui a besoin d'une spécificité

Les chemins

Les chemins de l'API

Nom de l'objet Chemin de l'API
Allergene environment.apiUrl+'/allergene/find/'
Article environment.apiUrl+'/article/find/'
ArticleCategorie environment.apiUrl+'/articleCategorie/find/'
Horaire environment.apiUrl+'/horaire/find/'
Identite environment.apiUrl+'/identites/find/'
Image environment.apiUrl+'/article/images/'
ImageCategorie environment.apiUrl+'/imageCategorie/find/'
ItemCarte environment.apiUrl+'/itemCarte/find/'
ItemCategorie environment.apiUrl+'/categorie/find/'
MetaTag environment.apiUrl+'/metaTag/find/'
OptionsItem environment.apiUrl+'/options/find/'
Promotion environment.apiUrl+'/promotions/find/'
Tva environment.apiUrl+'/tva/find/'
WebConfig environment.apiUrl+'/webConfig/find/'
Article environment.apiUrl+'/article/find/'
HomePage environment.apiUrl+'/homePage/get/'

Par les chemins indiqués on obtient toute les occurrences des objets spécifiés. Si on y ajoute l'id (ex:environment.apiUrl+'/{{objet}}/find/{{id}}) Alors on obtient uniquement l'objet ayant pour id.

Il y a une alternative a find c'est d'utiliser filter comme ceci environment.apiUrl+'/{{objet}}/filter/{{categorieName}}} <- le nom de la catégorie

Filter n'affichera que les éléments visibles et n'est que pour Image, Article, ItemCarte

L'objet Article contien un chemin findOnTop qui recherchera les article mis en avant

Les objets émis par filter sont dans le bon ordre

L'objet HomePage s'obtient avec une requête Get sur environment.apiUrl+'/gomePage/get/

Exemple d'une simple requête

const allergenesFindUrl = environment.apiUrl+'/allergene/find/';
test():void{
this.http.get(this.allergenesFindUrl).pipe(take(1)).subscribe(allergenes=>{
for(let allergene:allergenes)
{
console.log(allergene.name);
}
});
}
constructor(private http: HttpClient) { }

L'objet client

Pour obtenir l'objet client, contenant les imformations ainsi que le panier. Il faut faire une requete GET sur environment.apiUrl+'/client/get/{{idClient}}/{{securityKey}}'. Security key étant la clef reçue lors de l'authentication

Pour faire une mise a jour des imformations du client ou de son panier il faut faire une requête PUT avec dans le body un objet Client (json) a l'adresse environment.apiUrl+'/client/update/{securityKey} ou securityKey est la clef générée lors de la connexion

Pour la mise a jour si elle s'est bien déroulée on a en réponse (string) ok sinon (string) un message d'erreur
